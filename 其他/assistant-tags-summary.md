# 标签库总结

## 说明
- **输入标签**：对大总管的命令，需要向API发送的资料
- **输出标签**：对API的命令，需要API产生的内容
- **互动流程**：小助手 → 大总管 → API → 大总管 → 小助手

## 输入标签
- 故事设置：请提供故事的基本设定，包括背景、世界观、规则等信息
- 角色信息：请提供角色的详细信息，包括描述、性格、背景、关系、场景描写示例、创作者笔记、标签、物品、事件和数值
- 故事上下文：请提供之前的故事内容，用于保持剧情连贯性
- 所有故事内容：请提供生成的所有故事内容，用于参考角色在故事中的表现
- 最新故事内容：请提供最新生成的故事内容
- 当前故事内容：请提供当前的故事内容
- 角色描述：请提供用户输入的角色描述
- 角色数值：请提供角色的数值信息
- 日记设置：请提供用户设计的日记设置内容
- 当前时间：请提供当前游戏世界的时间信息
- 故事卡片：请提供故事卡片信息
- 临时角色库：请提供临时角色库信息
- 通用上下文：通用上下文信息
- 用户输入：用户输入内容
- 提示词：提示词内容
- 场景描述：场景描述
- 故事类型和风格设置：请提供故事的类型和风格设置，用于生成符合要求的故事内容
- 时间流逝速度：请提供时间流逝的速度设置
- 时间更新间隔：请提供时间更新的间隔设置
- 关键词：请提供用于检测特殊场景的关键词
- 数值配置：请提供角色数值的配置信息
- 玩法：请提供特殊场景的玩法信息
- 角色描述模板：请提供角色描述的模板信息
- 角色生成规则：请提供角色生成的规则信息
- 剧情建议模板：请提供剧情建议的模板信息
- 时间线索关键词和规则：请提供用于分析时间流逝的关键词和规则

## 输出标签
- 新故事内容：请根据故事设置、角色信息和故事上下文，生成新的故事内容，确保剧情连贯且符合角色人设
- 剧情建议：请基于当前故事内容和角色信息，提供详细的剧情发展建议，包括可能的情节走向和角色互动
- 角色日记：请以角色第一人称视角，根据故事内容生成真实生动的日记内容，反映角色的内心活动和情感变化
- 角色描述：请根据角色描述和故事内容，生成详细的角色描述，包括外貌特征、性格特点和行为方式
- 角色性格：请根据角色信息和故事表现，生成详细的角色性格特点，包括性格特质、行为方式和思维模式
- 角色背景：请根据角色信息和故事内容，生成完整的角色背景故事，包括成长经历、重要事件和影响因素
- 角色关系：请根据角色信息和故事内容，生成角色与其他角色的详细关系，包括关系类型、互动模式和发展历程
- 场景描写示例：请根据角色信息，生成关于角色的详细场景描写，包括环境细节、角色动作和氛围营造
- 创作者笔记：请根据角色信息和故事内容，生成关于角色的创作者笔记，包括设计思路、发展方向和注意事项
- 角色标签：请根据角色信息和故事表现，生成适合角色的标签，包括性格标签、身份标签和特征标签
- 角色物品：请根据角色信息和故事内容，生成角色相关的物品列表，包括物品名称、描述和与角色的关联
- 角色事件：请根据角色信息和故事内容，生成角色相关的事件列表，包括事件描述、发生时间和对角色的影响
- 角色数值：请根据角色信息，生成角色的初始数值信息，包括各项属性的具体数值
- 更新后角色数值：请根据剧情发展，更新角色的数值信息，确保数值变化符合剧情逻辑
- 更新后时间：请根据故事内容中的时间线索，更新游戏世界的时间信息，确保时间流逝合理
- 临时角色名称：请根据角色描述和故事内容，生成适合临时角色的名称，确保名称符合角色特点和背景
- 临时角色描述：请根据角色描述和故事内容，生成详细的临时角色描述，包括外貌特征、性格特点和行为方式
- 临时角色性格：请根据角色描述和故事表现，生成详细的临时角色性格特点，包括性格特质、行为方式和思维模式
- 临时角色标签：请根据角色描述和故事表现，生成适合临时角色的标签，包括性格标签、身份标签和特征标签
- 角色是否杀青：请根据故事内容，判断角色是否已经杀青，即是否在故事中完成了其角色使命
- 生成日记：请按照日记设置，根据所有的故事内容生成完整的故事日记，确保日记风格一致且内容连贯
- 最近记录：请保存最近生成的故事内容，用于快速访问和参考

## 互动流程详解
1. **小助手**：
   - 定义输入标签和输出标签
   - 构建提示词
   - 向大总管提交请求

2. **大总管**：
   - 接收小助手的请求
   - 处理输入队列，批量处理请求
   - 合并相同的请求
   - 根据输入标签收集内容
   - 构建增强提示词，包含所有小助手的需求
   - 调用API
   - 解析API响应
   - 确保所有输出标签都有内容
   - 根据输出标签分发给对应的小助手

3. **API**：
   - 接收大总管的请求
   - 调用AI服务（如DeepSeek）
   - 处理API响应
   - 返回响应给大总管

4. **小助手**：
   - 接收大总管分发的响应
   - 处理响应
   - 生成输出

## 注意事项
- 所有输入标签内容应详细、具体，以便API生成准确的内容
- 输出标签格式应严格按照要求，确保每个标签都有详细的对应内容
- 小助手应根据自身功能定义合适的输入标签和输出标签
- 大总管会自动合并相同的请求，提高API调用效率
- API响应会被解析为标签化格式，确保每个输出标签都有内容

## 标签使用示例
### 人设小助手 (ProfileAssistant)
- **输入标签**：角色信息、所有故事内容、当前故事内容
- **输出标签**：角色描述、角色性格、角色背景、角色关系、场景描写示例、创作者笔记、角色标签、角色物品、角色事件、角色数值、角色日记

### 时间小助手 (TimeAssistant)
- **输入标签**：当前时间、故事上下文、当前故事内容、时间流逝速度、时间更新间隔、时间线索关键词和规则
- **输出标签**：更新后时间

### 色色小助手 (EroticAssistant)
- **输入标签**：角色信息、故事上下文、当前故事内容、角色数值、关键词、数值配置、玩法
- **输出标签**：更新后角色数值、角色数值

### 人物小助手 (CharacterGeneratorAssistant)
- **输入标签**：角色描述、故事上下文、临时角色库、角色描述模板、角色生成规则
- **输出标签**：临时角色名称、临时角色描述、临时角色性格、临时角色标签、角色是否杀青

### 故事小助手 (StoryAssistant)
- **输入标签**：故事设置、角色信息、故事上下文、当前故事内容、故事类型和风格设置、玩法
- **输出标签**：新故事内容、剧情建议

### 总结小助手 (SummaryAssistant)
- **输入标签**：故事卡片、所有故事内容、日记设置
- **输出标签**：生成日记、最近记录

                        任务小助手计划书
================================================================================

一、显示形式
================================================================================

1. 任务面板入口
   - 在世界管理页面的侧边栏添加"任务系统"入口
   - 显示形式：带有图标的任务按钮，显示当前活跃任务数量

2. 任务中心页面 (task-assistant/index.html)
   - 顶部：当前目标展示区（显示自定义的总目标）
   - 中间：任务列表区
     * 活跃任务卡片（可展开查看详情）
     * 已完成任务列表
     * 失败/过期任务列表
   - 底部：奖励仓库和角色技能面板

3. 任务提示形式
   - 故事生成时：任务小助手会在故事内容中插入任务提示
   - 专门的提示气泡或标签：高亮显示任务相关内容和奖励

4. 奖励仓库面板
   - 网格形式展示获得的奖励物品
   - 显示物品图标、名称、效果说明
   - 点击可查看详情或使用

5. 角色技能面板
   - 分类展示已解锁的技能
   - 显示技能等级、冷却时间、效果

================================================================================

二、主要作用
================================================================================

1. 目标引导
   - 用户可以设定一个总体目标（如：让主角成为武林盟主）
   - 任务小助手根据目标拆分成多个子任务
   - 在故事进程中适时发布任务，推动剧情朝向目标发展

2. 任务管理
   - 自动生成与目标相关的任务
   - 任务类型：主线任务、支线任务、随机事件
   - 任务状态：未开始、进行中、已完成、已失败、已过期

3. 奖励系统
   - 完成任务获得奖励：经验值、金币、道具、技能
   - 奖励仓库：存储所有获得的奖励
   - 奖励可自定义风格（中式、西幻、现代等）
   - 奖励使用后从仓库移除

4. 惩罚系统
   - 任务失败或超时触发惩罚
   - 惩罚形式：属性下降、负面状态、失去道具、剧情倒退
   - 可自定义惩罚风格和严重程度

5. 技能系统
   - 奖励的技能存储在角色技能面板
   - 技能可主动使用（消耗冷却时间）
   - 技能效果：影响故事发展、增强主角能力、改变剧情走向

================================================================================

三、和其他小助手的互动
================================================================================

1. 与故事小助手的互动
   - 任务小助手 → 故事小助手：在故事中插入任务发布、完成、失败的内容
   - 故事小助手 → 任务小助手：根据剧情发展更新任务状态

2. 与角色小助手的互动
   - 任务小助手 → 角色小助手：根据任务需要生成相关角色
   - 角色小助手 → 任务小助手：角色相关任务触发（如：认识新角色）

3. 与人设小助手的互动
   - 任务小助手 → 人设小助手：根据任务需要调整角色设定
   - 人设小助手 → 任务小助手：角色设定变化可能触发相关任务

4. 与时间小助手的互动
   - 任务小助手 → 时间小助手：任务有时间限制，需要时间系统配合
   - 时间小助手 → 任务小助手：时间推进时检查任务是否超时

5. 与色色小助手的互动
   - 任务小助手 → 色色小助手：发布与情节相关的特殊任务
   - 色色小助手 → 任务小助手：完成特殊互动后更新任务进度

6. 与总结小助手的互动
   - 任务小助手 → 总结小助手：任务完成时生成任务总结
   - 总结小助手 → 任务小助手：根据总结内容评估任务完成情况

================================================================================

四、运行机制
================================================================================

1. 初始化阶段
   - 用户首次设定总体目标
   - 系统根据目标生成初始任务列表
   - 任务小助手注册到大总管

2. 任务生成机制
   - 主动生成：根据目标主动创建任务
   - 被动触发：根据其他小助手的行为生成关联任务
   - 随机事件：随机生成支线任务增加趣味性

3. 任务发布时机
   - 故事开始时：发布初始任务
   - 故事进行中：基于剧情发展发布新任务
   - 完成任务后：自动发布下一个关联任务
   - 随机触发：根据时间或事件概率发布随机任务

4. 任务进度追踪
   - 通过大总管的标签系统追踪任务相关的内容
   - 识别故事中与任务相关的描述
   - 自动判断任务完成或失败条件

5. 奖励发放
   - 完成任务 → 奖励生成 → 存入奖励仓库
   - 奖励可以是：道具、技能、属性加成、剧情道具
   - 奖励风格根据世界观自定义

6. 惩罚触发
   - 任务超时 → 触发惩罚
   - 任务失败 → 触发惩罚
   - 惩罚内容根据设置执行

7. 技能使用
   - 技能存储在角色技能面板
   - 用户可主动使用技能（消耗冷却）
   - 技能使用后效果体现在故事生成中
   - 使用后从仓库移除

8. 数据存储
   - 世界ID + task_assistant_前缀存储任务数据
   - 奖励仓库独立存储（可导出导入）
   - 技能面板与角色关联存储

================================================================================

五、自定义设置项
================================================================================

1. 目标设置
   - 总目标描述（文本输入）
   - 目标类型（成长型、收集型、剧情型、混合型）
   - 目标难度（简单、普通、困难、地狱）

2. 奖励设置
   - 奖励风格（武侠、仙侠、玄幻、西幻、现代、科幻）
   - 奖励类型偏好（道具、技能、属性、剧情）
   - 奖励稀有度设置

3. 惩罚设置
   - 惩罚风格（对应奖励风格）
   - 惩罚严重程度（轻微、中等、严重）
   - 是否启用惩罚

4. 任务设置
   - 同时进行的最大任务数
   - 任务刷新频率
   - 是否允许随机任务

5. 技能设置
   - 技能冷却时间
   - 技能效果强度

================================================================================

六、技术实现要点
================================================================================

1. 存储结构
   - tasks: 任务列表
   - rewards: 奖励仓库
   - skills: 角色技能面板
   - settings: 用户设置
   - currentGoal: 当前目标

2. 与大总管的交互
   - 输入标签：目标信息、当前剧情、角色状态、任务状态
   - 输出标签：任务内容、奖励内容、惩罚内容、技能内容

3. 文件结构
   - task-assistant/index.js: 主逻辑
   - task-assistant/settings.html: 设置页面
   - task-assistant/settings.js: 设置逻辑

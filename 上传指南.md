# Git 快速上传方法

## 方式一：使用 PowerShell 脚本（推荐）

创建一个 `upload.ps1` 文件：

```powershell
# 上传配置
$repoPath = "c:\Users\yf217\Desktop\aichat\开发\开发2.0（故事生成器）\3.2.0"
$commitMessage = "更新代码"

Set-Location $repoPath
git add .
git commit -m $commitMessage
git push origin master
```

运行命令：
```powershell
powershell -ExecutionPolicy Bypass -File upload.ps1
```

## 实际上传经验分享

### 更换远程仓库地址
当需要更换仓库地址时，可以使用以下命令：

```powershell
# 查看当前远程仓库地址
git remote -v

# 更新远程仓库地址
git remote set-url origin <新的仓库地址>

# 验证更新是否成功
git remote -v
```

### 处理文件变更
在上传前，建议先查看文件变更状态，确保所有需要上传的文件都已正确添加：

```powershell
# 查看文件状态
git status

# 添加所有变更文件
git add .

# 提交变更
git commit -m "更新说明"

# 推送到远程仓库
git push origin master
```

### 常见问题及解决方案

1. **提交信息包含空格或特殊字符**
   - 解决方案：使用单引号或双引号包裹提交信息
   - 示例：`git commit -m 'Update AI-story 5.0.0 files'`

2. **远程仓库不存在**
   - 解决方案：先在 GitHub 上创建仓库，然后再执行推送操作

3. **推送失败**
   - 解决方案：检查网络连接，确保 GitHub 仓库存在，必要时使用强制推送 `git push -f origin master`

### 推荐工作流程

1. 定期查看文件状态 `git status`
2. 有选择性地添加文件，避免提交不必要的文件
3. 编写清晰的提交信息，便于后续查看历史记录
4. 推送前确认远程仓库地址正确
5. 推送后在 GitHub 上验证代码是否成功上传
